# Poisson Consulting package CI system

Centrally-managed, parameterized GitHub Actions for the org's R packages.
CI logic lives in reusable workflows here in `poissonconsulting/.github`; each package carries
thin caller workflows that pass its classification.
Update a reusable workflow and every package inherits it on the next run.

## Components

Reusable workflows (`.github/workflows/`, `workflow_call`; callers must use `secrets: inherit`):

| Workflow | Inputs | Purpose |
| --- | --- | --- |
| `R-CMD-check.yml` | `tier`, `jags`, `private` | R CMD check; the tier selects the OS/R matrix and `error_on`. |
| `test-coverage.yml` | `private` | covr + Codecov. |
| `pkgdown.yml` | `private` | Build + deploy the docs site to `gh-pages`. |
| `check-no-suggests.yml` | `private` | R CMD check with hard dependencies only (CRAN packages). |

Each package's `.github/workflows/` holds matching thin callers (generated by the sync tool),
plus the unchanged fledge callers (`fledge-bump.yml`, `fledge-tag-on-merge.yml`).

Tooling:

- `tools/package-tiers.tsv` — the tier registry (source of truth for `tier`).
- `tools/sync-ci.sh` — renders callers from the registry + auto-detection and opens routed PRs.

## Classification

Per package, the sync tool resolves:

- **tier**: registry entry, else detected-on-CRAN -> `cran`, else `unimportant` (default).
- **private**: from repo visibility -> selects `GITHUB_PAT` (`PRIVATE_ACTIONS_PAT` for private repos so private dependencies install; `GITHUB_TOKEN` otherwise). This is the fix for the dependency-resolution failures the earlier flat rollout hit.
- **jags**: from DESCRIPTION (`rjags|runjags|jagsUI|R2jags`) -> adds the macOS/Windows JAGS installs (Linux gets JAGS via `setup-r-dependencies` system requirements).
- **cran**: `tier == cran` -> also gets the `check-no-suggests` caller.

Only `tier` is hand-maintained, and only to promote a non-CRAN package to `important` (or force a tier).
`private`, `jags`, and CRAN membership are auto-detected at sync time and never listed in the registry.

## Tier matrices (set inside `R-CMD-check.yml` from the `tier` input)

| tier | matrix | error_on | extra |
| --- | --- | --- | --- |
| cran | macOS-release, Windows-release, Ubuntu-{devel,release,oldrel-1} | warning | + check-no-suggests |
| important | macOS-release, Windows-release, Ubuntu-release, Ubuntu-oldrel-1 | warning | — |
| unimportant | Ubuntu-release | error | — |

## Registry

`tools/package-tiers.tsv`, one `<package> <tier>` per line (`#` comments).
List only packages to promote to `important` or to force a tier; everything else defaults to
`unimportant`, and CRAN packages auto-resolve to `cran`.

## Sync tool

```sh
tools/sync-ci.sh                    # dry run: classify every package + planned action
tools/sync-ci.sh --apply            # render callers + open routed PRs
tools/sync-ci.sh --apply pkgA pkgB  # restrict to named packages (piloting)
tools/sync-ci.sh --close-old        # close the prior f-standardize-actions PRs/issues/branches
ENGINE_REF=f-ci-reusable tools/sync-ci.sh --apply pkgA   # pin callers to a branch (pre-v1 pilot)
```

Routing follows the CODEOWNER: a normal PR when Joe owns the repo, otherwise an issue plus a
draft PR assigned to the owner.
The tool is dry-run by default, idempotent (skips repos with an open `f-ci` PR), throttled, and
isolates per-repo failures so one transient does not abort the batch.

Note: a very high burst of API calls can trip GitHub's secondary rate limiter, which makes a
dry run under-report (packages drop based on the throttle window). It clears within minutes;
re-run and it converges.

## Run sequence

1. Verify classification: `tools/sync-ci.sh` (full dry run) — expect ~79 packages, no `WARN`. Adjust `tools/package-tiers.tsv`.
2. Merge the reusable-workflow PR into `main`, then move the tag forward so callers can pin it:
   `git checkout main && git pull && git tag -f v1 && git push -f origin v1`.
3. Retire the previous flat rollout: `tools/sync-ci.sh --close-old`.
4. Pilot one package per class and watch CI (especially that private repos now resolve dependencies):
   `tools/sync-ci.sh --apply nlist embr jmbr poisslack baserowapi`.
5. Full rollout: `tools/sync-ci.sh --apply`; merge the green PRs, triage any genuine red checks per package.

## Prerequisites

- `PRIVATE_ACTIONS_PAT`, `CODECOV_TOKEN` org secrets available to the package repos (private repos already use both).
- Callers pin the reusable workflows at `@v1`; keep the `v1` tag pointing at the current `.github` main.

## Notes

- Moving `v1` also moves the fledge callers' pin, but the fledge reusable workflows are unchanged, so they keep working.
- A genuine package failure (e.g. tests needing live API credentials) stays red regardless of tier; that is a real bug to fix in the package, not a CI issue.
- The reusable-workflow matrix is built with `fromJSON(needs.matrix.outputs.config)`, a standard supported pattern.
